Chapter2 研究方向

挺认同这段话的

啃别人的代码比啃屎还难受啊！

已知/可用工具 etc

  1. 激活版010 Editor 主要是修改和对比特别方便
  2. GARbro Github开源项目 但是我的NET版本不够 vs生成不了… 还是借着另一位师傅的vs make的
  3. krkr官方xp3打包器 在KiriKiri2开发包下 (再次吐槽国内搜索引擎… 找个开源开发包都完全搜不到!!! 全是sb站的xxx文章)
  4. krkr-xp3 一个开源的XP3解包打包器 方便用来研究XP3文件格式 跟一些elf_parser很类似的功能

掌握的情报(…)

  1. XP3支持包含文件和目录
  2. XP3支持存储文件名
  3. XP3支持针对文件的压缩
  4. XP3支持基于文件的加密
  5. XP3支持对索引的压缩 (何解?)
  6. Nope…

接下来的一个思路就很棒了 将XP3文件与zip格式对比
可以大致知道这种压缩类文件的格式:
1.第一个文件头部
2.第一个文件
3.第二个文件头部
4.第二个文件
5.……
6.第n个文件头部
7.第n个文件
8.索引
9.索引尾部

逆过万华镜也知道 这种管理是比较高效的 也很好用结构体来实现

Chapter3 实验过程

这部分开始就是以前逆向时忽略的点了 压缩
目的: 确定XP3文件的性质和结构
先通过无加密无压缩的XP3搞清楚最基本的结构 再探索有压缩有加密的结构

压缩算法实验

确定XP3封包中使用的压缩方法
怎么确定呢?
将压缩后的 XP3 数据和使用 Bandizip 压缩后的数据进行对比，以确定压缩算法.

krkr-xp3的python代码中引入了zlib库
并且level设置的为9

结合zlib based on GNU zip 我们尝试bandzip以最大级别gz压缩一个test.txt
然后再用GARbro/官方 封包来封
